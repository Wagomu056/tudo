# Implementation Plan: tudo Kanban TUI

**Branch**: `001-kanban-tui` | **Date**: 2026-02-27 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/001-kanban-tui/spec.md`

## Summary

Build `tudo` — a single-binary Rust TUI application that presents a four-column
kanban board (Todo → Doing → Checking → Done) in the terminal using ratatui.
Task cards are navigated with vim-style keys; a right-side detail panel shows
the focused card's full text. Board state is persisted to `current.log` (JSON)
on every mutation; completed tasks are appended to `YYYYMMDD.log` (JSON Lines).
The Done column auto-clears on the next calendar day.

## Technical Context

**Language/Version**: Rust stable (≥ 1.75 via rustup)
**Primary Dependencies**: ratatui 0.28 (crossterm backend), serde + serde_json 1.0, chrono 0.4
**Storage**: Plain files — `current.log` (JSON, overwritten) and `YYYYMMDD.log` (JSON Lines, append-only)
**Testing**: `cargo test` — unit tests inline; integration tests in `tests/`
**Target Platform**: macOS and Linux terminal emulators; 80-column minimum width
**Project Type**: Single-binary TUI desktop application
**Performance Goals**: ≤ 16 ms per render frame (60 fps); keypress response < 100 ms
**Constraints**: ≤ 4 crate dependencies (excluding dev-deps); no async runtime; no binary data formats
**Scale/Scope**: Single user; up to ~200 tasks without visible lag

## Architecture Notes

### AppMode State Machine

```
Normal ──── a / e / E ────→ InputTitle / InputDetail
                               │
                    Enter / Esc│
                               ↓
                            Normal
```

### Render Flow (every frame)

```
terminal.draw(|frame| {
    let [cols_area, detail_area] = layout_horizontal(frame.area());
    let [col0, col1, col2, col3] = layout_columns(cols_area);
    render_column(frame, col0, &app, Status::Todo);
    render_column(frame, col1, &app, Status::Doing);
    render_column(frame, col2, &app, Status::Checking);
    render_column(frame, col3, &app, Status::Done);
    render_detail(frame, detail_area, &app);
    if app.mode != AppMode::Normal {
        render_input_popup(frame, frame.area(), &app);
    }
})
```

### Storage on Every Mutation

```
app.mutate(action)
  → storage::save_board(&app.board, "current.log")?
  → if action == MoveToDone {
        storage::append_done_entry(&entry, &filename)?
    }
  → app.status_msg = err.map(|e| e.to_string())
```

Errors surface in a one-line status bar at the bottom of the screen; they do
not propagate to a crash.

### Daily Done Filter (on startup)

```
board = storage::load_board("current.log")?;
let today = Local::now().date_naive();
board.tasks.retain(|t| {
    t.status != Status::Done
    || t.done_at.map(|d| d.date_naive() == today).unwrap_or(false)
});
storage::save_board(&board, "current.log")?;
```
